================================================================
  SPOTIFY CLONE — DOCUMENTAZIONE DEL PROGETTO
================================================================

PANORAMICA GENERALE
-------------------
Clone dell'interfaccia di Spotify, interamente frontend (nessun
backend). Usa l'API iTunes di Apple (gratuita, zero configurazione)
per cercare musica e riprodurre anteprime da 30 secondi.
Non richiede account Spotify né chiavi API.


================================================================
  STRUTTURA DEI FILE
================================================================

  index.html       → struttura HTML della pagina
  style.css        → tema scuro, layout, componenti visivi
  data.js          → funzioni utility condivise
  itunes-api.js    → chiamate all'API iTunes + dati statici
  app.js           → tutta la logica dell'applicazione
  documentazione.txt → questo file


================================================================
  index.html
================================================================

Contiene l'intera struttura statica della pagina, divisa in blocchi:

  SCHERMATA DI LOGIN (#loginScreen)
  - Overlay a schermo intero che copre tutto (z-index 9999)
  - Form con campo email/nome utente e password (decorativa)
  - Bottone ACCEDI con id="loginSubmitBtn" per la gestione async

  APP PRINCIPALE (#mainApp)
  - Nascosta di default (classe Bootstrap "d-none")
  - Viene mostrata dopo il login riuscito
  - Layout a griglia CSS: sidebar | area principale / player in basso

  SIDEBAR (.sidebar)
  - Logo cliccabile → torna alla Home
  - Navigazione principale: Home, Cerca, Libreria
  - Azioni: Crea playlist, Brani che ti piacciono
  - Lista playlist utente (#userPlaylists) — popolata da JS

  AREA PRINCIPALE (.main-content)
  - Barra in alto (.top-bar): pulsanti indietro/avanti, menu utente
  - Area contenuto (#contentArea): sostituita dinamicamente da JS
    ad ogni cambio di pagina

  PLAYER IN BASSO (.player)
  - Tre colonne: info brano | controlli + progress bar | volume
  - Elemento <audio id="audioPlayer"> nascosto gestisce l'audio HTML5

  MODALI BOOTSTRAP
  - #playlistModal  → crea una nuova playlist
  - #profileModal   → modifica nome e foto profilo

================================================================
  data.js
================================================================

Contiene tre funzioni utility caricate prima di tutto il resto,
disponibili globalmente in itunes-api.js e app.js.

  make(tag, props, ...children)
  - Helper DOM centrale del progetto.
  - Crea un elemento HTML con tag, attributi e figli in una riga.
  - Props supportate:
      class     → el.className
      text      → el.textContent (sicuro contro XSS)
      style     → stringa CSS o oggetto { proprietà: valore }
      onCLICK   → addEventListener('click', fn) (qualsiasi evento)
      altro     → el.setAttribute(k, v)
  - Accetta figli annidati (array, null, false ignorati)

  createCover(text, color1, color2)
  - Genera una copertina SVG con gradiente come fallback
    quando iTunes non ha immagini disponibili.
  - Restituisce una stringa data:image/svg+xml;base64 usabile in <img src>
  - Il testo (es. "IT", "Ch") viene centrato sull'SVG

  formatDuration(seconds)
  - Converte secondi interi in stringa "m:ss"
  - Es: 214 → "3:34"


================================================================
  itunes-api.js
================================================================

Gestisce tutto ciò che riguarda dati esterni e dati statici.

  COSTANTE: ITUNES_API
  - Base URL dell'API Apple: https://itunes.apple.com
  - Tutte le chiamate vengono fatte a questo dominio

  DATI STATICI: VIRTUAL_PLAYLISTS (array)
  - 8 playlist predefinite (Top Italia, Chill Vibes, Workout Mix…)
  - Ogni playlist ha: id, title, description, term (parola chiave
    usata per la ricerca iTunes), color1/color2 (per la copertina SVG)
  - Non contengono brani: i brani vengono cercati al momento
    dell'apertura tramite itunesGetPlaylistTracks()

  DATI STATICI: GENRES (array)
  - 12 generi musicali con nome, termine di ricerca e colore CSS
  - Mostrati nella pagina Cerca come griglia colorata

  itunesSearch(term, entity, limit)
  - Cerca su iTunes per termine, tipo ('song' o 'album') e limite
  - Aggiunge country:'it' per risultati italiani
  - Restituisce array di oggetti grezzi iTunes

  itunesGetAlbum(albumId)
  - Recupera un album specifico per ID con tutte le sue tracce
  - Restituisce { album, tracks } dove album è il primo risultato
    di tipo 'collection' e tracks sono i risultati di tipo 'track'

  itunesGetPlaylistTracks(playlistId)
  - Trova la playlist virtuale per id, legge il suo "term"
  - Chiama itunesSearch con quel termine e restituisce i brani

  normalizeTrack(item)
  - Converte un oggetto grezzo iTunes → oggetto traccia dell'app:
      id         → String(trackId)
      title      → trackName
      artist     → artistName
      album      → collectionName
      albumId    → String(collectionId)
      duration   → trackTimeMillis / 1000 (in secondi)
      cover      → artworkUrl100 ingrandita a 600x600
      previewUrl → previewUrl (null se assente → icona 🔇)

  normalizeAlbum(item)
  - Converte un oggetto grezzo iTunes → oggetto album dell'app:
      id, title, artist, cover, trackCount


================================================================
  app.js
================================================================

Cuore dell'applicazione. Contiene stato, navigazione, rendering
di ogni pagina e logica del player audio.

----------------------------------------------------------------
  STATO GLOBALE (oggetto `state`)
----------------------------------------------------------------
  displayName     → nome utente corrente
  profilePhoto    → foto profilo in base64 (da localStorage)
  currentTrack    → oggetto traccia in riproduzione
  currentPlaylist → array di tracce della coda corrente
  currentIndex    → indice della traccia corrente nella coda
  isPlaying       → booleano riproduzione attiva
  isShuffle       → booleano shuffle attivo
  isRepeat        → booleano repeat attivo
  volume          → numero 0–1 (default 0.7)
  likedTracks     → Map<id, TrackObject> (persistita in localStorage)
  userPlaylists   → array [{ id, name, tracks }] (persistita)
  history         → array di pagine visitate (per indietro/avanti)
  historyIndex    → posizione corrente nella history

----------------------------------------------------------------
  VARIABILI GLOBALI
----------------------------------------------------------------
  audio           → elemento <audio id="audioPlayer">
  _trackRegistry  → Map<id, TrackObject>: registro di tutti i brani
                    resi visibili nell'interfaccia; usato da toggleLike
                    e playTrackInList per trovare i brani senza
                    passarli come parametri
  _cache          → oggetto chiave→Promise: evita di chiamare
                    la stessa API due volte nella sessione

----------------------------------------------------------------
  AVVIO (DOMContentLoaded)
----------------------------------------------------------------
  - Collega handleLogin al submit del form
  - Se localStorage contiene 'display_name' (sessione precedente)
    → salta il login, carica i dati e avvia l'app direttamente

----------------------------------------------------------------
  LOGIN (handleLogin — async)
----------------------------------------------------------------
  1. Legge il valore del campo email/nome utente
  2. Valida: campo vuoto o email malformata → mostra errore rosso
  3. Disabilita il bottone e mostra spinner Bootstrap
  4. Attende 800ms (simulazione chiamata async — sostituibile
     con un fetch() a un backend reale)
  5. Salva displayName in stato e localStorage
  6. Chiama loadPersistedData(), nasconde login, mostra app
  7. Chiama initApp()

----------------------------------------------------------------
  PERSISTENZA localStorage
----------------------------------------------------------------
  loadPersistedData()  → legge liked_tracks, user_playlists,
                         profile_photo e li mette nello stato
  saveLikedTracks()    → serializza Map → array di coppie [id, obj]
  saveUserPlaylists()  → serializza array playlist

----------------------------------------------------------------
  INIT APP (initApp)
----------------------------------------------------------------
  Eseguito una volta dopo login. In ordine:
  1. updateUserMenu()      → mostra nome utente e foto in header
  2. setupPlayer()         → aggancia tutti gli event listener del player
  3. setupNavigation()     → aggancia click su nav-link, back, forward…
  4. renderUserPlaylists() → popola la sidebar con le playlist utente
  5. showPage('home')      → carica la Home

----------------------------------------------------------------
  NAVIGAZIONE
----------------------------------------------------------------
  navigateTo(page, pushHistory)
  - Aggiunge la pagina alla history (come la history del browser)
  - Chiama showPage() e aggiorna il link attivo nella sidebar

  goBack() / goForward()
  - Scorrono l'array history tramite historyIndex
  - Non fanno nuove chiamate API se la pagina è già in _cache

  showPage(page) — async
  - Smista la stringa "page" alle funzioni di rendering corrette:
      'home'              → renderHome()
      'search'            → renderSearch()
      'library'           → renderLibrary()
      'liked'             → renderLikedTracks()
      'profile'           → renderProfile()
      'album-{id}'        → renderAlbum()
      'playlist-{id}'     → renderPlaylist()
      'userplaylist-{id}' → renderUserPlaylist()
      'genre-{name}'      → renderGenre()
  - Mostra spinner durante il caricamento
  - Cattura errori e mostra messaggio generico

----------------------------------------------------------------
  PAGINE — FUNZIONI DI RENDERING
----------------------------------------------------------------

  renderHome(container)
  - Saluto dinamico (Buongiorno / Buon pomeriggio / Buonasera)
  - Quick-grid: prime 6 VIRTUAL_PLAYLISTS come card orizzontali
    con bottone play diretto (senza aprire la playlist)
  - Card-grid: tutte le VIRTUAL_PLAYLISTS come card verticali
  - Card-grid album: top albums da iTunes (ricerca 'top hits')

  renderSearch(container)
  - Input di ricerca con debounce 400ms (evita troppe chiamate API)
  - Griglia generi (GENRES) sempre visibile
  - performSearch() popola il div risultati con brani e album

  renderGenre(container, genreName)
  - Cerca i brani per il termine del genere scelto
  - Mostra titolo + tracklist

  renderLibrary(container)
  - Mostra le playlist utente create (se presenti)
  - Mostra le prime 4 playlist virtuali come "in evidenza"

  renderAlbum(container, albumId)
  - Recupera album + tracce da iTunes tramite itunesGetAlbum()
  - Header con copertina grande, nome artista, numero brani
  - Bottone play grande → avvia tutte le tracce
  - Tracklist con numeri di traccia

  renderPlaylist(container, playlistId)
  - Trova la playlist virtuale nei VIRTUAL_PLAYLISTS
  - Recupera i brani con itunesGetPlaylistTracks()
  - Stessa struttura dell'album ma con copertina SVG generata

  renderUserPlaylist(container, playlistId)
  - Trova la playlist nelle userPlaylists dello stato
  - Mostra i brani aggiunti dall'utente (o messaggio se vuota)

  renderLikedTracks(container)
  - Legge la Map likedTracks dallo stato
  - Header con icona cuore, nome utente, conteggio brani
  - Se vuota mostra messaggio di invito a salvare brani

  renderProfile(container)
  - Avatar (foto o iniziale con gradiente)
  - Nome, conteggio playlist e brani salvati
  - Bottone "Modifica profilo" → apre modal
  - Bottone logout → cancella localStorage e torna al login
  - Griglia delle playlist utente

----------------------------------------------------------------
  HELPER DOM
----------------------------------------------------------------

  makeCard(cover, title, description, page)
  - Crea una card album/playlist cliccabile
  - Al click apre la pagina corrispondente
  - Il bottone play verde in hover usa quickPlay()

  quickPlay(page)
  - Avvia la riproduzione da una card senza aprire la pagina:
      album-{id}    → recupera tracce e chiama playTracksList()
      playlist-{id} → chiama playPlaylistById()

  makePlaylistHeader(coverSrc, type, title, ...metaNodes)
  - Costruisce l'header grande (copertina 232px + info)
  - coverSrc = null → usa copertina con icona cuore (liked tracks)

  makeTrackRow(track, index, ids, showAlbumCol)
  - Crea una singola riga della tracklist
  - Colonna numero: mostra icona volume se è la traccia in play
  - Icona 🔇 se previewUrl è null
  - Bottone cuore per il like
  - Doppio click → playTrackInList(track.id, ids)

  renderTrackList(trackList, showAlbumCol)
  - Registra tutti i brani nel _trackRegistry
  - Costruisce l'header della tabella + righe con makeTrackRow()

----------------------------------------------------------------
  PLAYER AUDIO
----------------------------------------------------------------

  setupPlayer()
  - Aggancia tutti gli event listener del player al DOM:
      playBtn          → togglePlay()
      prevBtn          → prevTrack()
      nextBtn          → nextTrack()
      shuffleBtn       → toggleShuffle()
      repeatBtn        → toggleRepeat()
      likeBtn          → toggleLike(currentTrack.id)
      progressContainer → seek sull'audio
      volumeContainer  → imposta volume e aggiorna barra
      volumeBtn        → muto/senza muto (toggle 0 ↔ 0.7)
  - audio 'timeupdate' → aggiorna barra di avanzamento e tempo
  - audio 'ended'      → repeat o nextTrack()

  playTrack(track)
  - Aggiorna tutto il DOM del player (titolo, artista, copertina)
  - Se previewUrl esiste: imposta audio.src e chiama audio.play()
  - Se previewUrl è null: mostra info ma non riproduce
  - Dopo il play: aggiorna bottone, aggiorna pagina corrente
    (per mostrare l'icona volume nella riga)

  playTracksList(trackList)
  - Imposta currentPlaylist e currentIndex = 0
  - Chiama playTrack() sulla prima traccia

  playTrackInList(trackId, ids)
  - Ricostruisce la playlist dai _trackRegistry usando gli ids
  - Trova l'indice del brano cliccato e chiama playTrack()
  - In questo modo il tasto "avanti" continua nella stessa lista

  togglePlay()
  - Se in riproduzione → pausa
  - Se in pausa (con previewUrl) → riprende e refreshCurrentPage()
  - Se nessun brano → non fa nulla

  prevTrack()
  - Se la traccia è oltre i 3 secondi → torna all'inizio
  - Altrimenti → traccia precedente nella coda (con wrap)

  nextTrack()
  - Shuffle attivo → indice casuale
  - Altrimenti → traccia successiva (con wrap)

  toggleShuffle() / toggleRepeat()
  - Invertono il booleano nello stato
  - Aggiungono/rimuovono la classe CSS 'active' sul bottone

  updateVolumeIcon()
  - Muto → bi-volume-mute-fill
  - < 50% → bi-volume-down-fill
  - ≥ 50% → bi-volume-up-fill

----------------------------------------------------------------
  LIKE
----------------------------------------------------------------

  toggleLike(trackId)
  - Se già in likedTracks → rimuove
  - Se non presente → cerca il brano in _trackRegistry o nel
    currentTrack e lo aggiunge alla Map
  - Salva su localStorage, aggiorna icona cuore nel player,
    ricarica la pagina corrente

  updateLikeBtn()
  - Aggiorna l'icona cuore nel player (pieno/vuoto, verde/grigio)
    in base allo stato di likedTracks per il brano corrente

----------------------------------------------------------------
  PLAYLIST UTENTE
----------------------------------------------------------------

  createPlaylist()
  - Legge il nome dal modal, crea oggetto { id:'up'+timestamp, name, tracks:[] }
  - Aggiunge a userPlaylists, salva su localStorage, chiude modal,
    aggiorna la sidebar

  renderUserPlaylists()
  - Svuota e ricrea i link nella sidebar (#userPlaylists)

----------------------------------------------------------------
  PROFILO
----------------------------------------------------------------

  openProfileModal()
  - Salva la foto corrente in tempProfilePhoto (per annullare)
  - Precompila il campo nome, mostra anteprima foto

  previewProfilePhoto(e)
  - Legge il file immagine con FileReader e lo converte in base64
  - Aggiorna l'anteprima nel modal senza salvare ancora

  saveProfile()
  - Salva nome e foto (base64) in stato e localStorage
  - Aggiorna l'avatar nell'header, chiude il modal, ricarica la pagina

  updateUserMenu()
  - Aggiorna testo nome utente nell'header
  - Se c'è una foto: sostituisce l'icona <i> con un <img>
  - Se non c'è foto: ripristina l'icona <i>

  logout()
  - Rimuove display_name e profile_photo da localStorage
  - Ferma l'audio, resetta lo stato, mostra la schermata di login


================================================================
  FLUSSO TIPICO DI UNA SESSIONE
================================================================

  1. L'utente apre index.html
  2. Se non ha mai fatto login → vede la schermata di login
     → inserisce email → async login con spinner → entra nell'app
  3. La Home carica le playlist virtuali e gli album da iTunes
  4. L'utente clicca su una playlist → renderPlaylist() scarica
     i brani, li mostra in tracklist
  5. Doppio click su una traccia → playTrackInList() avvia l'audio
  6. Il player in basso aggiorna titolo/artista/copertina e progress bar
  7. Clic sul cuore → toggleLike() salva su localStorage
  8. Al prossimo accesso: auto-login, brani salvati già presenti


================================================================
  LIMITI TECNICI NOTI
================================================================

  - Anteprime da 30 secondi: iTunes API fornisce solo preview,
    non i brani completi. Le tracce senza preview mostrano 🔇.

  - Nessun backend: utenti, password e playlist esistono solo
    nel localStorage del browser corrente.

  - Cache in memoria: _cache si azzera ad ogni ricarica pagina.
    Non c'è cache persistente per i risultati API.

  - file:// protocol: alcuni browser bloccano l'audio se il file
    viene aperto direttamente (doppio click). Usare un server
    locale (es. Live Server di VS Code, oppure: npx serve .).

  - CORS: l'API iTunes supporta CORS, ma alcune CDN Apple per
    l'audio potrebbero avere restrizioni in certi browser.
